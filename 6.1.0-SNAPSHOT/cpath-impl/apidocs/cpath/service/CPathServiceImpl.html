      • topPathways

        public ServiceResponse topPathways(String[] organisms,
                          String[] datasources)
        Gets top (root) pathways (URIs, names) in the current BioPAX model. "Top pathway" can mean different things... 1) One may want simply collect pathways which are not values of any BioPAX property (i.e., a "graph-theoretic" approach, used by ModelUtils.getRootElements(org.biopax.paxtools.model.Model, Class) method) and by BioPAX normalizer, which (in the cPath2 "premerge" stage), for all Entities, generates relationship xrefs to their "parent" pathways. 2) Another approach would be to check whether specific (inverse) properties, such as controlledOf, pathwayComponentOf and stepProcessOf, are empty. Here we follow the second method.
        Specified by:
        topPathways in interface CPathService
        Parameters:
        organisms - filter values (URIs, names, or taxonomy IDs)
        datasources - filter values (URIs, names)
        Returns:

      • map

        public Set<String> map(String fromDb,
              String fromId,
              String toDb)
        Description copied from interface: CPathService
        Maps an identifier to primary ID(s) of a given type. Normally, the result set contains only one ID. If the result contains more than one value, which does not necessarily an error, then it's up to other methods to decide how to proceed; e.g., one should not probably merge different data objects if the mapping is known to be umbiguous, but it's usually ok to generate relationship xrefs or use the resulting IDs in a BioPAX graph query.
        Specified by:
        map in interface CPathService
        Parameters:
        fromDb - data collection name or null (to use all source ID types)
        fromId - the source ID
        toDb - standard (MIRIAM) preferred name of the target ID type (e.g., 'UniProt')
        Returns:
        a set of primary IDs of the type; normally one or none elements
