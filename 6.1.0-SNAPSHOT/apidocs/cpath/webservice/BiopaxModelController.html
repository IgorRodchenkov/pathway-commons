      • cpathIdInfo

        @RequestMapping(method=GET,
                value="/{localId}")
public void cpathIdInfo(@PathVariable
                              String localId,
                              Writer writer,
                              javax.servlet.http.HttpServletRequest request,
                              javax.servlet.http.HttpServletResponse response)
                 throws Exception
        This is a convenience method is to make cpath2 data more suitable for LinkedData / Semantic Web, by resolving cpath2-generated URIs (created in the data warehouse and during validation and normalization). Ideally, all URIs are to be resolvable URLs. Normally, one should use #elementById(Get, BindingResult, Writer, HttpServletRequest, HttpServletResponse).
        Parameters:
        localId - - the part of URI following xml:base TODO return a summary page (type, name, some numbers and links to get the biopax/sif/gsea data) instead of the BioPAX content.
